## 2. 修复的问题

### 2.1 登录后“网络异常 / CORS error”
- 现象：`login` 成功，但 `info/codes` 报 CORS 或预检失败。
- 修复：
  - CORS 放行 `OPTIONS` 方法。
  - Sa-Token 拦截器中对 `OPTIONS` 预检请求直接放行。
  - 补齐兼容接口 `GET /auth/info`（复用已有 `/user/info`）。

### 2.2 前端登录 404 / 环境变量缺失
- 现象：登录请求路径异常、`VITE_APP_TITLE` 未定义、API 地址缺失导致请求错误。
- 修复：
  - 新建 `apps/web-antd/.env.development`。
  - 配置：
    - `VITE_APP_TITLE=AIO-LIFE`
    - `VITE_GLOB_API_URL=http://localhost:45678/api`

### 2.3 MinIO 图片不显示（CBTI 头像破图）
- 现象：页面大量图片占位符，后端取图 404/连接失败。
- 修复：
  - 启动 MinIO（本机 `minio.exe`，端口 `1300/1301`）。
  - 重启后端触发 CBTI 初始化上传。
  - 验证图片预览 URL 返回 `200 image/png`。

### 2.4 权限菜单“删除”能力下线
- 移除前端删除按钮与调用。
- 移除后端删除接口与对应 service 方法，避免被绕过调用。

## 3. 修改原因

- 菜单启用状态应成为“可运营配置”，而不是仅靠前端静态路由控制。
- 登录链路不稳定主要是“接口路径不一致 + 预检被拦截”，需要统一与兼容。
- CBTI 结果页需要完整分享闭环（文案、海报、下载、扫码回流），提升可传播性。
- MinIO 属于项目核心依赖（对象存储），未启动会直接影响图片链路，需补齐启动与文档说明。
- README 需要反映当前 fork 事实和真实可执行的本地启动方式，降低交接成本。

## 4. 技术实现细节

### 4.1 菜单状态更新
- Controller 新增状态专用接口，避免全量更新校验冲突。
- Service 新增 `updateStatus`，仅更新 `status` 与公共更新字段。
- 前端列表行内开关变更后刷新树数据，保证管理页与数据一致。

### 4.2 CBTI 海报生成
- 前端 Canvas 方案：
  - 背景渐变、人格信息、匹配度、优势/注意、二维码等元素手绘。
  - 图片生成后使用 DataURL 预览与下载。
- 复制文案：
  - 优先 `navigator.clipboard.writeText`
  - 降级 `textarea + execCommand('copy')`

### 4.3 认证与跨域
- CORS 配置允许 `OPTIONS`，减少浏览器预检失败。
- Sa-Token 对预检请求短路放行，避免 `NotLoginException` 打断跨域流程。
- 补 `/auth/info` 兼容接口，解决前后端路径历史差异。

### 4.4 依赖与启动
- 补齐前端运行依赖（含 `qrcode`、`@types/qrcode` 及相关缺失包）。
- 前后端端口联调：
  - 前端：`5173`
  - 后端：`45678`
  - MinIO：`1300/1301`

## 5. 遇到的坑

- 预检请求报 500 时，前端通常只显示 CORS/网络异常，根因在后端拦截器日志中。
- 登录成功不代表登录链路完整成功，`/auth/info` 与 `/auth/codes` 仍需单独验证。
- MinIO 未启动时，业务接口可能正常但图片链路全部失败，容易误判为前端缓存问题。
- monorepo 下依赖解析受 workspace 影响，需在实际入口应用包补齐直接依赖。
